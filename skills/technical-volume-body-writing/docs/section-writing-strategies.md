# 章节写作策略

本文件用于根据技术卷章节标题选择正文写作重点。

## 需求分析类

适用标题关键词：

- 需求分析
- 项目理解
- 现状分析
- 建设背景
- 业务理解
- 系统现状
- 数据现状

写作重点：

1. 招标人业务背景和建设目标；
2. 当前信息化、生产系统、数据资源、接口、流程、网络、安全、运维现状；
3. 数据汇聚、系统集成、业务协同、质量管理等方面的痛点；
4. 后续方案设计依据；
5. 需求边界和实施关注点。

推荐结构：

1. paragraph：总体说明当前章节分析目的；
2. paragraph：结合招标要求描述现状和目标；
3. list：归纳关键需求或痛点；
4. paragraph：说明对后续方案设计的支撑作用。

避免：

1. 直接写实施步骤；
2. 编造招标人已有系统名称；
3. 空泛描述“信息化水平较高”；
4. 脱离招标文件要求写行业背景。

## 总体技术方案类

适用标题关键词：

- 总体技术方案
- 总体架构
- 技术路线
- 系统设计
- 功能设计
- 数据架构
- 接口方案

写作重点：

1. 总体建设思路；
2. 技术架构和模块划分；
3. 数据采集、汇聚、治理、共享、应用流程；
4. 系统集成、接口、安全、性能、扩展性；
5. 方案与评分标准、招标要求之间的对应关系。

推荐结构：

1. paragraph：总体建设思路；
2. list：关键技术措施或功能模块；
3. paragraph：说明架构可扩展性、可维护性和安全性；
4. table：必要时展示模块、职责、成果。

避免：

1. 只写“先进、可靠、高效”等口号；
2. 没有具体技术措施；
3. 未回应关键评分点；
4. 编造不存在的平台或产品能力。

## 实施方案类

适用标题关键词：

- 实施方案
- 实施计划
- 项目组织
- 进度安排
- 交付计划
- 联调测试
- 验收方案

写作重点：

1. 实施阶段划分；
2. 每个阶段的任务、成果和责任；
3. 需求确认、方案设计、开发配置、测试联调、试运行、验收交付；
4. 进度控制、沟通机制、风险控制；
5. 与招标人协同配合方式。

推荐结构：

1. paragraph：总体实施原则；
2. list：阶段任务和交付成果；
3. paragraph：过程管控和风险应对。

避免：

1. 只有流程名，没有任务内容；
2. 不说明交付成果；
3. 不说明协同配合；
4. 不说明质量和进度控制。

## 保障措施类

适用标题关键词：

- 质量保障
- 进度保障
- 安全保障
- 风险控制
- 组织保障
- 运维保障

写作重点：

1. 管理制度和组织责任；
2. 检查、评审、测试、验收机制；
3. 问题发现、记录、整改、复核、关闭闭环；
4. 风险识别、预警、处置和复盘；
5. 过程记录和交付物质量控制。

推荐结构：

1. paragraph：保障体系总体说明；
2. list：保障措施；
3. paragraph：闭环管理和交付质量说明。

避免：

1. 只写“加强管理”；
2. 缺少检查机制；
3. 缺少问题闭环；
4. 缺少责任落实。

## 售后服务类

适用标题关键词：

- 售后服务
- 运维服务
- 培训服务
- 服务响应
- 持续优化

写作重点：

1. 服务响应机制；
2. 问题受理、分析、处理、回访闭环；
3. 培训支持和知识转移；
4. 运维保障和持续优化；
5. 服务记录、服务质量和客户满意度管理。

推荐结构：

1. paragraph：服务目标和服务体系；
2. list：响应、处理、培训、优化措施；
3. paragraph：持续服务和保障效果。

避免：

1. 编造固定响应时间，除非招标文件或知识库明确提供；
2. 编造服务团队人数；
3. 编造本地化服务机构；
4. 只写承诺，不写处理流程。
