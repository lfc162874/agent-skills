# 禁止内容规则

本文件定义技术卷正文生成时必须避免的格式、编号、事实和表达问题。

## 禁止 Markdown

正文文本字段中不得出现以下 Markdown 符号或格式：

1. `**` 加粗标记；
2. `#` 标题标记；
3. `-`、`*`、`+` 行首列表标记；
4. `>` 引用标记；
5. 三个反引号代码块；
6. Markdown 表格分隔符；
7. Markdown 链接语法。

如需分点表达，必须使用 `list` block。

## 禁止手写编号

正文不得手写以下编号：

1. `1. 标题`；
2. `1、标题`；
3. `一、标题`；
4. `（一）标题`；
5. `(一) 标题`；
6. `第一，`、`首先，`、`其次，`、`再次，`连续构成伪编号时不得使用；
7. `1.1`、`1.1.1` 等章节编号。

章节编号由大纲树生成，列表编号由前端或后端渲染器生成。

## 禁止用空格控制格式

不得在 `text` 或 `items` 字段中使用以下方式控制格式：

1. 段首普通空格；
2. 段首全角空格；
3. 制表符；
4. 多个连续空格；
5. 使用换行模拟段落布局。

段落缩进、行距、字体、字号由 `styleKey` 和渲染器控制。

## 禁止大纲越界

不得生成以下内容：

1. 当前章节之外的一级、二级、三级标题；
2. 商务卷、资格卷、报价文件内容；
3. 与当前章节无关的行业介绍；
4. 未经允许的小标题；
5. 重复当前章节标题作为正文第一段。

## 禁止无证据事实

没有 `knowledgeContext` 明确支撑时，不得写：

1. 企业资质等级；
2. 证书名称和证书编号；
3. 软件著作权、专利、奖项；
4. 项目案例名称；
5. 客户名称；
6. 合同金额、中标金额；
7. 人员姓名、职称、证书；
8. 固定响应时间、服务网点、团队人数；
9. 平台产品名称和具体功能清单；
10. 任何看似具体但无证据来源的数字。

证据不足时，应使用通用能力表达，不能虚构事实。

## 禁止低质量表达

应避免以下表达：

1. “高度重视”“全力以赴”“确保万无一失”等空泛口号；
2. 大量重复“本项目”“我公司将”；
3. 只表态不说明动作；
4. 只写目标不写措施；
5. 只写流程名不写交付物；
6. 与评分标准没有关系的套话；
7. “以下是”“综上所述”等非正文提示语；
8. 暴露 AI 生成痕迹的表达。
