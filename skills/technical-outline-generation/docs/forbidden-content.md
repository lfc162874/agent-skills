# 技术卷禁用内容规则

## 1. 总体原则

技术卷不得混入商务卷、资格卷、报价文件内容。

除非招标文件明确要求相关内容放入技术卷，否则以下内容不得出现在技术卷章节中。

## 2. 禁止作为技术卷章节的内容

以下内容不得作为技术卷一级、二级或三级章节：

1. 营业执照；
2. 法定代表人身份证明；
3. 法定代表人授权委托书；
4. 投标函；
5. 开标一览表；
6. 报价表；
7. 分项报价表；
8. 商务偏离表；
9. 资格审查资料；
10. 财务报表；
11. 审计报告；
12. 纳税证明；
13. 社保证明；
14. 企业资质证书；
15. 项目业绩证明；
16. 合同复印件；
17. 中标通知书；
18. 信用中国截图；
19. 无重大违法声明；
20. 保证金凭证。

## 3. 禁止生成的通用模板章节

除非评分标准、技术要求或采购需求中存在明确依据，否则不得生成以下泛化章节：

1. 项目背景；
2. 项目概述；
3. 总体方案；
4. 技术路线；
5. 项目管理方案；
6. 质量保障；
7. 风险控制；
8. 应急预案；
9. 售后服务；
10. 培训方案；
11. 服务承诺；
12. 组织保障。

如果确实生成这些章节，必须在 `basis` 或 `evidenceRefs` 中说明证据来源。

## 4. 禁止行为

生成技术卷大纲时不得：

1. 抄招标文件目录作为技术卷目录；
2. 使用无证据支撑的万能目录；
3. 把商务、资格、报价内容混入技术卷；
4. 自造采购需求中不存在的模块名称；
5. 为了凑章节数量生成重复章节；
6. 使用和项目无关的行业模板；
7. 忽略高分评分项；
8. 输出无法解析的 JSON；
9. 输出 Markdown 代码块；
10. 输出推导过程或解释文字。
